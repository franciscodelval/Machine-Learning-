FUNCIÓN SIGMOIDEA LOGÍSTICA¶

Ahora aplicamos la regresion logstica para entrenar un modelo, para encontrar el limite de decision optimo y hacer prediccion de clases. El objetivo de la regresin logstica es minimizar la funcin de costo

1) Funcin sigmoidea

Esta funcin es la "activacin" para la salida de la regresin logstica. Convierte una entrada continua en un valor entre cero y uno. Este valor se puede interpretar como la probabilidad de la clase o la probabilidad de que el ejemplo de entrada se clasifique positivamente. Usando esta probabilidad junto con un valor umbral, podemos obtener una prediccin de etiqueta discreta.

In [136]:
def sigmoid(x):
    return 1 / (1 + np.exp(-x))

2) Funcin de costo

La funcin de costo evala el rendimiento del modelo en los datos de entrenamiento dado un conjunto de parmetros del modelo. Aqu est la funcin de costo para la regresin logstica.

In [138]:
def compute_cost(theta, x, y): #la funcion de costo depende de los parametros theta, x e y 
    m = len(y)  # creamos un obejto 'm' para ver el numero de observaciones=20
    y_pred = sigmoid(np.dot(x , theta)) # y = sigma(h^0 * x)
    error = (y * np.log(y_pred)) + ((1 - y) * np.log(1 - y_pred)) #error (toda la formula completa )
    cost = -1 / m * sum(error) # ahora aadimos el costo 
    gradient = 1 / m * np.dot(x.transpose(), (y_pred - y)) # utilizamos la funcion gradiente para unir las funciones anteriores
    return cost[0] , gradient

Se reduce la salida a un nico valor escalar, que es la suma del "error" cuantificado como una funcin de la diferencia entre la probabilidad de clase asignada por el modelo y la etiqueta verdadera. La implementacin est completamente vectorizada: calcula las predicciones del modelo para todo el conjunto de datos en una declaracin (sigmoide (X * theta)).

4) Funcin Clculo Gradiente

Ahora escribimos una funcin que calcule el gradiente de los parmetros para saber cmo cambiar los parmetros para poder mejorar el resultado del modelo en el entrenamiento.

Con el descenso de gradiente no solo cambiamos aleatoriamente los valores de los parmetros para ver qu funciona mejor. En cada iteracin de entrenamiento, actualizamos los parmetros de una manera que est garantizada para moverlos en una direccin que reduce el error de entrenamiento (costo). Podemos hacer esto porque la funcin de costo es diferenciable.

In [140]:
def gradient_descent(x, y, theta, alpha, iterations): # descenso de gradiente esta en funcion de x, y, theta, alpha, iteraciones
    costs = [] 
    for i in range(iterations): # hacemos un bucle para iterarse un numero de veces 
        cost, gradient = compute_cost(theta, x, y) # a medida que se baja en el gradiente, se minima el coste 
        theta -= (alpha * gradient) # Alpha: tasa de aprendizaje del algoritmo. se multiplica * gradiente en funcion de theta 
        costs.append(cost) # creamos una unica lista
    return theta, costs # devuelveme theta , costs
